## Quick Reference

### Key Architectural Principles

1. **Description-First:** Store semantic descriptions, not video (privacy + efficiency)
2. **Event-Driven:** Motion triggers AI processing (cost optimization)
3. **Real-Time Updates:** WebSocket notifications for live dashboard
4. **Multi-Provider AI:** Automatic fallback for reliability
5. **Zero-Config Database:** SQLite for plug-and-play deployment
6. **Single Camera MVP:** Designed to scale to multi-camera in Phase 2

### Technology Choices

- **Frontend:** Next.js 15 + TypeScript + Tailwind CSS + shadcn/ui
- **Backend:** FastAPI + Python 3.11+ + SQLAlchemy + OpenCV
- **Database:** SQLite (MVP) → PostgreSQL (Phase 2)
- **AI Models:** OpenAI GPT-4o-mini (primary), Gemini Flash (fallback), Claude Haiku (tertiary)
- **Real-Time:** WebSocket (FastAPI/Starlette built-in)
